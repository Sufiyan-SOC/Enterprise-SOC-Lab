# T1543.003 - Windows Service

## Technique

Create or Modify System Process: Windows Service

## Tactic

Persistence

Privilege Escalation

## Lab Scenario

A Windows service was created using **sc.exe** to simulate persistence behavior commonly observed during post-exploitation.

## Detection

- Microsoft Sysmon Event ID 1

## SIEM Detection

Splunk Enterprise detected execution of the Windows service creation command.

## Investigation Data

- User
- Endpoint
- Process
- Parent Process
- Command Line
- Timeline

## Mitigation

- Restrict service creation permissions.
- Monitor **sc.exe** execution.
- Investigate unauthorized services.
- Correlate service creation with other administrative activity.

## Detection Outcome

The simulated Windows service creation activity was successfully detected, investigated, documented, and mapped to MITRE ATT&CK.
