# T1087.001 - Local Account Discovery

## Technique

Local Account Discovery

## Tactic

Discovery

## Lab Scenario

Simulated post-compromise reconnaissance on a Windows 11 Enterprise endpoint. The attacker executed native Windows discovery commands to enumerate the current user, local user accounts, group memberships, and local administrators after obtaining access to the system.

## Detection

- Microsoft Sysmon Event ID: 1 (Process Creation)

Monitored Processes:

- whoami.exe
- net.exe

## SIEM Detection

Splunk Enterprise was configured to detect execution of Windows native account and group discovery utilities using Microsoft Sysmon Process Creation events.

## Investigation Data

- Endpoint Name
- Username
- Executed Process
- Command Line
- Parent Process
- Process Execution Timeline

## MITRE ATT&CK Mapping

| Tactic | Technique | ID |
|---------|-----------|----|

| Discovery | Local Group Discovery | T1069.001 |

## Attack Commands

```cmd
whoami

hostname

whoami /groups

net user

net localgroup administrators
```

## Mitigation

- Monitor execution of Windows discovery commands.
- Enable Microsoft Sysmon Process Creation logging.
- Restrict unnecessary local administrator privileges.
- Correlate discovery activity with privilege escalation and lateral movement events.
- Investigate repeated discovery commands executed by non-administrative users.

## Detection Outcome

The simulated discovery activity was successfully captured by Microsoft Sysmon, forwarded to Splunk Enterprise, and detected using custom SPL queries. The generated alert was investigated through process telemetry, timeline analysis, and dashboard visualization within the Enterprise SOC Lab.
